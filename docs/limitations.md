# Limitations

## Minimal public sample layer

This public repo ships only a minimal sample layer.

That means some workflows are easier to understand from the docs than to run at full scale without the private companion data repo.

## Private operating data is separate

The full working corpus, complete source lists, and private generated outputs live in `chess-data-private`, not here.

## Example opening families

Stonewall, French, and Habits are presented as example repertoires / example study families.

They help demonstrate the workflows, but they are not meant to imply that the code only works for one setup.

## PDF transition state

The long-term direction is markdown-first documentation in GitHub, with PDFs treated as optional secondary exports.

Some generator scripts are still present, but PDFs are not the canonical document layer.

## Fair-play boundary

This repo is for study, reporting, and retrospective analysis.

It is not for:
- live move assistance
- correspondence cheating
- hiding engine use
